# Contributing to Radius

Thanks for your interest in contributing! Here's how to get involved.

## How to Contribute

### Reporting Bugs
Open an issue on GitHub and include:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Browser and OS

### Suggesting Features
Open an issue with the `enhancement` label. Describe the use case and why it would benefit office teams.

### Submitting Code

1. Fork the repo
2. Create a branch: `git checkout -b feature/your-feature-name`
3. Make your changes in `index.html`
4. Test in at least one browser with a real Google Maps API key
5. Open a pull request with a clear description of what you changed and why

## Code Style

- Keep everything in `index.html` (single-file architecture)
- Use vanilla JS — no frameworks
- Follow the existing CSS variable system for theming (`--primary`, `--surface`, etc.)
- Comment non-obvious logic

## Questions?

Open an issue and tag it `question`.
